Bonne pratique de code


“ Un langage de programmation est bas niveau
quand son programme nécessite de faire attention
à ce qui n’est pas pertinent.”
Alan Jay Perlis

L'ergonomie du développement , Liaison entre efficacité et lisibilité

Dans le monde du développement il est important de considérer tous points d’un programme que l’on développe. Cette etudes complète va permettre une mise à utilisation, ou en production optimale.
Ci dessous une liste non exhaustive de points à prendre en comptes, qui permettront un code optimisé.

  • 01

    Une documentation
    complète :

    Une documentation complète est importante, puisqu'elle va permettre à l’utilisateur de s’y retrouver dans les fonctions que propose un programme, ce qui permettra d’améliorer son expérience client et l’efficacité de son usage.

  • 02

    Efficacité/optimisation :

    Un code optimisé va être par conséquent, plus efficace, il consiste de manière concrète à réduire au maximum les calculs superflus, la déclaration de variable pouvant être remplacée, et donc d'accélérer le processus du programme.

  • 03

    La lisibilité du code :

    Un code lisible est important pour permettre la modification rapide en cas de besoin et l’amélioration probable, de plus en cas de travail collaboratif (ce qui est souvent le cas), cela permet une meilleure compréhension des autres informaticiens.


  • 04

    Simplicité d’utilisation :

    Ce point est constitué par plusieurs autres, que ce sont la documentation (ancre sur la doc) et l’ergonomie du programme fournie. (interface graphique)

  • 05

    Fiabilité/robustesse :

    Cela va comprendre la prise en charge des erreurs, la réduction du nombre de problèmes dans le code, sa sécurisation aussi, ne pas négliger les failles potentielles.


  • 06

    Adaptabilité : plateforme/version

    Ce point est séparé en plusieurs points, tout d’abord l’extensibilité, c'est-à-dire la capacité pour un programme d’être améliorable plus ou moins facilement, et donc que les outils requis pour l’implémentation de mise à jour soit d’une difficulté modérée.

    Enfin, il peut être défini par la capacité d'adapter son code (fonction, bibliothèque utilisée, langage) à la plateforme sur laquelle il est destiné. Par exemple, un langage tel que python va être assez pratique pour programmer des logiciels pour PC ou le machine learning , tandis que pour le développement mobile, il sera assez voire très médiocre, on privilégiera donc le Java par exemple.

    Enfin, on peut aussi évoquer le C++ assez polyvalent puisqu’il permet de programmer des jeux vidéo, des systèmes embarqués pour des montres connectées par exemple ou encore des programmes de serveurs ou de micro-contrôleurs.
